# MVP Roadmap (Next)

本文件是基于当前实现状态的“后续推进计划”，聚焦可交付与可运维。

## Milestone A: 稳定性门禁收口（优先级 P0）
目标：把现有 smoke/gate 从“可运行”提升到“可决策发布”。

1. 扩展长稳用例
- 覆盖：连续运行、周期重启、断网重连、重复消息、异常 payload。
- 产物：结构化结果（JSON）+ 阈值判断。

2. 接入发布门禁
- 将稳定性关键阈值纳入 `release_candidate.py` 默认路径。
- 失败即阻断 RC，避免人工兜底。

验收标准：
- 稳定性 gate 在 CI/本地均可重复执行。
- 关键阈值（失败率、恢复时间）有明确红线。

## Milestone B: 测试网外部试用闭环（优先级 P0）
目标：让非开发用户按文档一次走通。

1. 一次性安装验证
- macOS/Windows 新机器安装。
- profile 切换到官方测试网并完成连通性检查。

2. 交易闭环验证
- 创建/导入钱包。
- faucet + send + history 状态一致。

验收标准：
- 按 `doc/INSTALLATION.md` + `doc/USER_QUICKSTART.md`，可在 30 分钟内完成。
- 至少一轮外部试用反馈闭环。

## Milestone C: 发布标准化（优先级 P1）
目标：降低发版不确定性。

1. RC 模板固定化
- 发布说明模板。
- 已知风险模板。
- 回滚记录模板。

2. 观测最小集固化
- 节点在线率、交易成功率、确认延迟、错误码分布。

验收标准：
- 每次 RC 都能自动生成同构报告。
- Runbook 能支持演练并复现。

## 非目标（当前阶段不做）
- 主网发布。
- 开放式无许可节点加入。
- 移动端钱包与复杂 GUI 重构。
- 外部安全审计作为上线前置。
